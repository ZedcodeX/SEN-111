WEEK 1

Web Development Overview

What is Web Development?

Web development, also known as website development, refers to the tasks associated with creating, building, and maintaining websites and web applications that run online on a browser. It includes web design, web programming, and database management.

Types of Web Development

Front-end Development

Focuses on the portion of the website users see and interact with. Front-end developers translate design into code using HTML, CSS, JavaScript, and frameworks like React, Angular, and Bootstrap.

Back-end Development

Focuses on how a website works behind the scenes. Handles database interactions, user authentication, server configuration, and business logic using languages like PHP, Ruby, Python, and tools like MySQL and Git.

Full-stack Development

Combines both front-end and back-end web development, allowing developers to work on all aspects of web applications.

Types of Websites

Static Websites

Content remains the same for all users unless code is changed. Written with HTML, CSS, and JavaScript without databases. Commonly used for individual profile pages.

Dynamic Websites

Content can change based on user interaction or time without code changes. Use databases and server-side languages like PHP, JavaScript, Python, C++, or Java. Common in social media, e-commerce, and banking.

Requirements for Web Development

Hardware Requirements

  • Server: Central computer with web applications and shared resources, processes requests and serves responses to clients
  • Client: User's computer or device for browsing, typically with a web browser installed

Software Requirements

  • Web Browser: Software for accessing website content (Chrome, Firefox, Edge, Safari, Opera)
  • HTML: Hypertext Markup Language for rendering website content
  • CSS: Cascading Style Sheets for designing the appearance of HTML objects
  • JavaScript: Provides interactivity between web page objects
  • Server-side Scripts: Languages like PHP, Python for database operations and server-side logic
  • Text Editor: Tools like Notepad++ or Sublime Text for writing code
WEEK 2

HTML Document Structure

Introduction to HTML

HTML (Hypertext Markup Language) is a markup language that uses tags to create objects on the browser. It is not a programming language but a rendering language, as it cannot perform logical operations.

Important: HTML objects are recreated each time a web page is loaded or reloaded. This is why you might lose page content when refreshing without a reliable internet connection.

HTML File Structure

HTML files have a .html or .htm extension. The basic structure of every HTML document follows this convention:

<!DOCTYPE HTML>
<html>
    <head>
        <!-- Information about the HTML file -->
    </head>
    <body>
        <!-- Actual content of the web page -->
    </body>
</html>

Understanding HTML Tags

Opening Tags

Made up of < character, tag name, element definitions, and > character. Examples include:

  • <p> - Paragraph
  • <b> - Bold
  • <input> - Input field
  • <table> - Table

Closing Tags

Made up of </ followed by tag name and >. The / indicates to the browser that this is a closing tag.

Commonly Used HTML Tags

<title> Tag

Sets the page title displayed in the browser's title bar, essential for navigation and SEO.

<p> Tag

Defines paragraphs. Text must be wrapped in paragraph tags to display correctly as paragraphs.

<div> Tag

One of the most used tags for grouping elements. Can dynamically receive HTML content via JavaScript using innerHTML.

<form> Tag

Groups input fields that users fill out and submit to the server. Includes method (GET/POST) and action attributes.

Text Formatting Tags

  • <b> or <strong> - Bold text
  • <i> or <em> - Italic text
  • <small> - Smaller font size
  • <br> - Line break
  • <sup> - Superscript
  • <sub> - Subscript

Grouping Tags

  • <span> - Groups inline text for styling
  • <section> - Groups related content
  • <div> - Generic container for block-level content
  • <fieldset> and <legend> - Groups form elements with a title

The Form Tag and HTTP Methods

GET Method

Used for non-confidential data. Data is attached to the URL with variables visible to third parties:

www.mysite.com/example.php?variable1=value1&variable2=value2

Note: Always use comments in your code! It's good programming practice and will help you understand your code when you return to it later.

WEEK 3

Cascading Style Sheets (CSS)

What is CSS?

CSS is a style sheet language used for describing the presentation of documents written in markup languages like HTML, XML, SVG, XHTML, or MathML. It is neither a programming language nor a markup language, but describes how elements should be rendered on screen, paper, speech, or other media.

CSS Code Structure

CSS code is written as groups of declarations that define property values for elements or groups of elements. For example:

p {
    color: red;
}

This code turns all paragraph text red in the HTML document.

Modes of Using CSS

Inline CSS

CSS set as the value of the style property directly on HTML elements. Quick but leads to tight coupling and code repetition.

Internal CSS

CSS written inside <style> tags within the HTML document. Reusable within the document but not across multiple pages.

External CSS

CSS written in a separate .css file and linked via <link> tag. Best practice - reusable, cacheable, and maintainable.

Example: Inline CSS

<div style="width: 10px; height: 20px; background-color: blue;"></div>

Example: External CSS

<link href="styles/style.css" rel="stylesheet" />

CSS Selectors

Selector Type What It Selects Example
Element Selector All HTML elements of specified type p selects all <p>
ID Selector Element with specific ID (unique) #my-id
Class Selector Elements with specified class .my-class
Attribute Selector Elements with specified attribute img[src]
Pseudo-class Selector Element in specified state a:hover

Common CSS Properties

  • margin: Space between an element and its neighbors
  • padding: Space between an element and its contents
  • border: Line marking the edges of an element
  • color: Font color of text
  • font-size: Size of text
  • font-family: Font styles for text (with fallback options)
  • background: Color or image in element's background
  • height: Element height
  • width: Element width

Best Practice: Use external CSS files for production websites. They can be cached by browsers, reducing load times and bandwidth consumption while making maintenance easier.

WEEK 6

JavaScript Fundamentals

Introduction to JavaScript

JavaScript is a client-side programming language popularly used in web applications. While HTML provides structure, JavaScript provides the logic and interactivity. JavaScript runs in the client's browser and controls client-side logic including:

  • Mathematical operations (simple to complex)
  • Controlling HTML element behaviors
  • Dynamically creating and modifying HTML elements and CSS properties
  • Communicating with remote servers

Programming Concepts

Programming is the use of structured computer logic to solve human problems. A computer program is a collection of code that solves a particular problem, while software is an organized collection of programs solving similar problems.

Variables

In JavaScript, variables are declared using the var keyword, followed by the variable name and optional initial value:

var firstName = 'Ephraim';

Naming Convention: Use camelCase for variable names (e.g., firstName, userEmail). Variable names can be alphanumeric but cannot begin with numbers or special characters (except underscore).

Data Types

Literals

Values assigned to variables can be:

  • String: Alphanumeric characters in quotes - 'yes1' or "yes1"
  • Numeric: Numbers without quotes - 42, 3.14

Arrays

Ordered collections of values, each with a zero-based index:

var houses = ["one storey", "1 Bedroom flat", "2 Bedroom flat"];
alert(houses[1]); // Returns "1 Bedroom flat"

Objects

Arrays with defined indices (keys or properties):

var myObject = {
    TenantName: "Michael Chuk",
    State: 'Kaduna',
    Country: 'Nigeria',
    address: 'No.7, Wanakamuna Street, Lagos',
    2: 'Number is allowed as key'
};

Functions

Collections of code lines that solve a defined problem. Functions can accept arguments and return values:

function nameOfFunction(argument1, argument2) {
    // Do something
    return something;
}

Linking Functions to HTML Events

Functions can be connected to HTML element actions:

<button onclick="myFunction()">Click Me</button>

Comparison Operators

== (Equal to)

Returns true if values are equal: 2 == 2 → true

!= (Not equal to)

Returns true if values differ: 2 != 3 → true

> (Greater than)

Returns true if left > right: 3 > 2 → true

< (Less than)

Returns true if left < right: 8 < 9 → true

Conditional Statements

IF Statement

if (condition) {
    // Code executed if true
} else {
    // Code executed if false
}

IF...ELSE IF Statement

if (condition A) {
    // Code for A
} else if (condition B) {
    // Code for B
} else {
    // Default code
}

Loops

WHILE Loop

while (condition) {
    // Code block
}

FOR Loop

for (var i = 0; i < 10; i++) {
    // Code block
}

FOR...IN Loop

for (var key in myObject) {
    alert(key + ': ' + myObject[key]);
}

Output Methods

  • console.log() - Output to browser console for debugging
  • alert() - Display message in popup dialog
  • prompt() - Request user input

Comments: Use // for single-line comments and /* */ for multi-line comments. Good programmers comment their code extensively!

WEEK 7

DOM Manipulation & AJAX

HTML DOM (Document Object Model)

The HTML DOM provides the means to access each HTML element and interact with its attributes. The document is accessed using the JavaScript keyword document.

Accessing HTML Elements

Get Element by ID

var elem = document.getElementById('elementIdHere');

Get Elements by Class Name

Returns a collection of elements. Access individual elements using array indices:

var elements = document.getElementsByClassName('myClass');
var firstElement = elements[0];

Get Elements by Tag Name

var inputs = document.getElementsByTagName('input');

Manipulating Element Properties

Getting and Setting Attributes

// Get value
var value = document.getElementById('myInput').value;

// Set value
document.getElementById('myInput').value = 'New Value';

Getting and Setting HTML Content

// Get innerHTML
var content = document.getElementById('myDiv').innerHTML;

// Set innerHTML
document.getElementById('myDiv').innerHTML = '<p>New content</p>';

Getting ID and Class

var elementId = elem.id;
var elementClass = elem.className;

AJAX (Asynchronous JavaScript)

AJAX allows communication with the server in the background without reloading the entire page. Benefits include:

  • Background server communication
  • Partial page updates (only affected sections reload)
  • Reduced bandwidth consumption
  • Concurrent server communications

AJAX Request Structure

// Create AJAX object
var xmlhttp;
if (window.XMLHttpRequest) {
    xmlhttp = new XMLHttpRequest(); // Modern browsers
} else {
    xmlhttp = new ActiveXObject("Microsoft.XMLHTTP"); // IE6, IE5
}

// Handle state changes
xmlhttp.onreadystatechange = function() {
    if (this.readyState == 4 && this.status == 200) {
        // Successful response
        var response = this.responseText;
    } else if (this.readyState == 4 && this.status != 200) {
        // Handle errors
    }
};

AJAX GET Request

xmlhttp.open("GET", "server.php?var1=value1&var2=value2", true);
xmlhttp.send();

Tip: To prevent caching of GET requests, append a dummy parameter with timestamp: '&dummy=' + Date.now()

AJAX POST Request

// Create FormData
var formData = new FormData();
formData.append('variable1', 'value1');
formData.append('variable2', 'value2');

xmlhttp.open("POST", "server.php", true);
xmlhttp.send(formData);

Dynamic Image Loading

JavaScript can load images dynamically using the FileReader API:

var reader = new FileReader();

reader.onload = function(data) {
    document.getElementById('myImage').src = data.target.result;
};

reader.readAsDataURL(imageFile.files[0]);
